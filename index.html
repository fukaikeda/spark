Designing SparkGPT: Accelerating Solar Development through Conversational AI

Section 1: Background

Solar and battery developers spend significant time gathering critical information like zoning rules, permitting hurdles, and historical records. In the United States, there are 3000+ counties, 10,000+ building permit jurisdictions and more than 2,000 energy-regulating entities - each with their own set of rules [Source: SparkAI]. Spark AI currently compiles this data into static reports, which are executive summaries with useful information for developers.

However, these reports can't capture every data point, and manually navigating them remains inefficient. To resolve this issue, our team was asked to design SparkGPT — a new interactive search feature inside of Spark's web platform that coexists with the report. Using SparkGPT, users can ask nuanced questions and quickly receive verified, cited answers, bridging the gap between raw data and the high level insights captured in the report.

Section 2: Project Goals

  • Explore Design Directions
    • Brainstorm and evaluate multiple interfaces for SparkGPT, experimenting with how it could integrate into Spark AI's existing report system
  • Build Interactive Prototypes
    • Create a basic low-fidelity prototype that outlines the basic visual components of the feature
    • Create an interactive high-fidelity prototype that demonstrates how users could engage with SparkGPT to extract meaningful insights
  • Support Source Transparency
    • Ensure every SparkGPT response includes clear, clickable citations that direct a user to the original, raw source (e.g., zoning document, meeting record, article)
  • Balance Functionality and Usability
    • Prototype a feature that meets both functionality and usability standards, making it intuitive and efficient for solar developers under time constraints

Section 3: Early Exploration

With these goals in mind, we sketched four possible designs for the SparkGPT feature.

Key Ideas

  • Two possible user workflows:
    • (1) SparkGPT is integrated into the Reports feature, and each chat is linked to a specific report.
    • (2) SparkGPT is a standalone feature on the SparkAI webpage.
  • Incorporating a side-by-side or top-bottom view of both the chat and the report
  • Allowing users to dig deeper into citations sourced for SparkGPT's response

Design Sketch Justifications

Home Screen

  • Explored two design philosophies:
    • Design #2: Information-dense, landing page style
    • Designs #1 & #4: Modern, minimalist aesthetic (aligned with current SparkAI design)
  • Group discussed whether SparkGPT warranted a full homepage redesign or a lighter touch.
  • Final decision: Iterative enhancement using the minimalist design for smoother integration.

Chat Screen

  • Continued the modern minimalist approach seen in leading AI chat interfaces.
  • Design aligned with success metrics from the client’s blurb:
    • Users should be able to query and receive actionable information to inform energy project decisions.
  • Included suggestion buttons for prompting example questions and guiding user interaction.

Expanded Chat Screen

  • Created in response to client feedback emphasizing source transparency.
  • Introduced split-screen layout for comparing SparkGPT responses with original source documents.
  • Design #3 innovation: A small search button on the Reports page to launch SparkGPT.
  • Client later noted this could cause confusion (seeming like a generic search tool instead of a chat interface).
  • These iterations helped highlight which sketches best advanced the product vision.

Section 4: Low-Fi Prototype

Next, we consolidated our sketches into three wireframe screens.

For our final version of the Menu Viewer wireframe, we took elements from Design 3 and decided to have SparkGPT be an integrated feature in the Reports section rather than a standalone feature so that chats can be linked to specific reports. The Report & Chat wireframe took inspiration from designs 1, 3, and 4, implementing some variation of side-by-side reports and chat view, which would allow users to scan through the report and ask questions simultaneously. Lastly, the Citation & Chat wireframe combined both Design 2 and Design 3, which highlighted the importance of linking to SparkGPT's answers to the original citations.

Section 5: High-Fi Prototype

Major Iterations Made:

We presented the previous wireframes in a live session with SparkAI and received actionable feedback, especially around citation clarity and integration points. Based on feedback we received from the Spark AI team, we made the following key adjustments to our design for the final, high-fidelity prototype:

  • Two Entry Points:
    • 1. From the Homepage
      • Users click “SparkGPT,” pick a report (e.g. Lancaster, PA), and ask questions
    • 2. From a Report Page
      • Users click a chat icon to open SparkGPT next to their current report
  • Searchable Chat Log Feature
    • Added a search bar for SparkGPT histories, helping users find previous answers quickly
  • Citation Transparency
    • Highlighted the corresponding text of the raw source directly instead of vague source links
